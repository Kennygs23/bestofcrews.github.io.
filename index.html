Git Launch Protocol

You fired the code into GitHub. Now let's connect the pipeline to bestofcrews.com.

WHY THIS IS BEST: Connecting GitHub to Netlify gives you automatic, encrypted SSL security, custom domain bindings, and instantly deploys every time you push changes to your GitHub main branch.
Step 01 // Authenticate Host

Login to Netlify via GitHub

Go to app.netlify.com. Instead of creating a regular email account, click "Sign up with GitHub". This instantly links your cloud workspace to your code archives.

Step 02 // Import Repository

Import from Git

On your Netlify dashboard, click the big button that says "Add new site" and select "Import an existing project". Choose GitHub, authorize permissions, and select the repository where you just fired your code.

Step 03 // Deploy Project

Build settings

Since this is a clean static site with no complex framework, leave all the build settings, commands, and folders **completely blank**. Just click "Deploy Project". Netlify will read your index.html and generate a live secure link.

Step 04 // Point Domain

Connect bestofcrews.com

Go to your new site's settings on Netlify, select "Domain Management", and enter bestofcrews.com. Put Netlify's DNS name servers into your registrar (GoDaddy, Namecheap, etc.) to seal the handshake.

ABOUT GH-PAGES: This is GitHub's native hosting. It is fast and free, but configuring custom domains with SSL takes a few more manual parameters in your DNS dashboard than Netlify.
Step 01 // Repository Check

Ensure Index File Naming

Make sure your primary code file in your repository is named exactly index.html (all lowercase) and is sitting directly in the root directory, not hidden inside any sub-folder.

Step 02 // Access settings

Open the Pages Panel

Open your repository on GitHub. Look at the tab navigation at the top of the repository page, click "Settings" (the gear icon), and select "Pages" from the left sidebar menu.

Step 03 // Choose Source

Select Main Branch

Under "Build and deployment", click the dropdown under "Branch". Change it from "None" to your primary branch (usually "main" or "master"), select "/ (root)", and hit "Save".

Step 04 // Set Domain

Input Custom Domain

Scroll down on the same Pages setting screen to the "Custom domain" input box. Type in bestofcrews.com and click save. Point your registrar DNS using an A-Record targeting GitHub's IP blocks to complete setup.
